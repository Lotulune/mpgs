# MPGS Context

MPGS is a multiplayer game discovery context centered on a maintained public discovery service and lightweight clients used by friends or other players.

## Language

**公共发现服务**:
由项目维护者运行的共享服务，负责提供已经整理好的多人游戏发现结果和相关分析。一个 **公共发现服务** 可以服务多个 **使用者客户端**。
_Avoid_: 每用户自建服务、本地一体服务、个人导库服务

**使用者客户端**:
使用者安装并打开的应用入口，用来连接一个 **公共发现服务** 并浏览、筛选、标记游戏。一个 **使用者客户端** 在第一版只连接一个 **公共发现服务**，且不内置默认服务地址。
_Avoid_: 本地后端、个人服务器、导库工具

**客户端壳能力**:
**使用者客户端** 保留的本地能力，包括保存服务地址、验证服务身份信息、保存个人游戏状态、打开外部链接和可选公共数据缓存。它不包含 Steam 同步、游戏发现、AI 分析或公共游戏库写入。
_Avoid_: 本地导库能力、客户端扫描能力、桌面后端

**客户端公共 API 调用**:
**客户端界面** 直接通过 HTTPS REST 请求读取 **公共发现服务** 的匿名只读 API。Tauri Rust 侧只处理客户端壳能力，不代理公共库读取请求。
_Avoid_: Tauri HTTP 代理、客户端 Rust 后端、通过命令读取公共库

**公共库只读缓存**:
**使用者客户端** 保存的最近一次公共游戏库读模型快照，用于服务不可达时弱离线浏览。它不能产生新的公共数据，也不能替代 **公共发现服务**。
_Avoid_: 本地公共库、离线导库、客户端同步源

**公共读缓存协商**:
**公共发现服务** 和 **使用者客户端** 通过 ETag 与条件请求减少公共库只读数据重复传输的机制。它服务于公共库只读缓存，不表示客户端拥有公共库写入权。
_Avoid_: 客户端同步协议、离线写入、无条件全量拉取

**公共库修订号**:
**公共发现服务** 用于生成公共读列表缓存标识的公开数据版本。列表 ETag 基于查询参数和公共库修订号，详情 ETag 基于单个游戏及其分析更新时间。
_Avoid_: 每条列表逐项哈希、客户端生成 revision、用缓存替代审核状态

**公共修订触发**:
会改变匿名只读 API 结果的公共库变更，例如公开字段、公开分析、可见性、排序分数或公开图片 URL 变化。仅运维、管理备注或不可见候选变化不触发公共库修订号递增。
_Avoid_: 管理动作全量递增、不可见状态影响公共缓存、任务内部进度触发 revision




**客户端本地存储**:
**使用者客户端** 使用的本地持久化空间，只保存服务地址、服务身份验证结果、个人游戏状态和公共库只读缓存。它不保存 Steam Key、LLM Key、发现任务或公共库写模型。
_Avoid_: 客户端公共库、客户端服务端数据库、本地导库数据库

**当前服务连接**:
**使用者客户端** 当前正在使用的唯一公共发现服务连接。第一版不聚合多个服务的数据。
_Avoid_: 多服务聚合、同时连接多个公共库、跨服务合并

**最近服务历史**:
**使用者客户端** 保存的少量历史服务连接记录，用于切换当前服务连接。个人游戏状态和公共库只读缓存必须按服务实例隔离。
_Avoid_: 共享收藏状态、跨服务缓存混用、服务身份不区分

**服务地址迁移**:
**服务实例 ID** 相同但服务地址变化时，使用者客户端更新该服务连接的地址和元信息，并沿用该服务实例下的个人游戏状态。
_Avoid_: 当作新服务、丢弃个人状态、按 URL 隔离状态

**服务实例分叉**:
服务实例名相同但 **服务实例 ID** 不同时，使用者客户端必须视为不同服务连接。个人游戏状态不能按实例名自动合并。
_Avoid_: 按名称合并、跨实例状态污染、实例 ID 可忽略



**服务地址**:
使用者客户端连接 **公共发现服务** 所需的域名或 IP 地址。第一版必须由使用者或维护者提供，不存在默认官方服务地址。
_Avoid_: 官方默认地址、硬编码后端、自动发现地址

**服务连接文件**:
由 **管理界面** 生成并提供给使用者的无密钥连接描述文件，包含服务名称、服务地址和期望 API 版本。导入服务连接文件不能跳过 **服务身份信息** 验证。
_Avoid_: 登录凭据、内置默认地址、密钥文件

**连接分享入口**:
**管理界面** 提供给维护者分享服务地址的入口，包括服务连接文件下载、复制服务地址，并预留二维码或协议链接体验。连接分享入口不能包含密钥或管理凭据。
_Avoid_: 分享 admin token、分享 setup token、带密钥二维码

**服务身份信息**:
**公共发现服务** 暴露给 **使用者客户端** 的公开识别信息，用来证明该地址是兼容的 MPGS 服务并声明 API 版本、实例名称和可用能力。**服务地址** 必须通过服务身份信息验证后才能保存或使用。
_Avoid_: 登录信息、管理员配置、健康检查文本

**服务能力标记**:
**服务身份信息** 中声明当前服务版本、API 版本、公共库状态和可用能力的公开字段。使用者客户端用它判断兼容性和界面可用性。
_Avoid_: 隐式能力推断、只返回 ok、客户端硬编码服务能力

**客户端 API 兼容性**:
**使用者客户端** 对公共发现服务 API 版本的支持范围。第一版客户端只支持 `v1`，遇到缺失或不匹配的 API 版本时拒绝保存服务地址。
_Avoid_: 隐式兼容、多版本分支、保存后运行时报错

**项目发布版本**:
MPGS 客户端和公共发现服务共享的发布版本号。同一个 GitHub Release 发布对应版本的客户端安装包、服务端镜像和相关部署资产。
_Avoid_: 客户端独立版本、服务端独立版本、用发布版本替代 API 版本


**服务实例名**:
维护者配置的 **公共发现服务** 公开显示名称，用于服务身份信息、服务连接文件和客户端连接状态。它不是认证信息，可由管理界面热加载更新。
_Avoid_: 域名、管理员身份、服务密钥

**服务实例 ID**:
**公共发现服务** 首次配置时生成的稳定公开 UUID，用于服务身份信息、服务连接文件和使用者客户端本地状态隔离。它不是秘密，也不能随每次启动变化。
_Avoid_: 服务地址、认证令牌、随机启动 ID

**服务地址可用性验证**:
**使用者客户端** 保存服务地址前执行的验证流程，必须同时读取服务身份信息并成功调用至少一个匿名只读公共 API。它验证 API 版本、HTTPS 或局域网规则，以及当前客户端环境下的跨源读取可用性。
_Avoid_: 只 ping health、跳过 CORS 检查、保存后再报错

**管理界面**:
项目维护者用来管理 **公共发现服务** 内容和运行状态的界面。它面向维护者，不是普通使用者的首次配置入口。
_Avoid_: 用户配置页、客户端设置页

**管理概览**:
**管理界面** 的维护者首页，展示服务状态、配置状态、公共库规模、待审核数量、最近任务、失败摘要、重启需求和连接分享入口。
_Avoid_: 发现首页、普通用户首页、个人收藏页

**管理概览 API**:
**管理访问** 下提供管理概览数据的独立接口，例如 `/api/v1/admin/overview`。它不能复用匿名只读的发现首页 API。
_Avoid_: 公共 discovery-home、匿名管理概览、前端拼多个内部接口

**部署诊断区**:
**管理界面** 中面向维护者的轻量诊断区域，用于展示 HTTPS、public base URL、Steam、LLM、R2、Postgres migration、restart policy 和 CORS 等配置健康状态。
_Avoid_: 普通客户端诊断、密钥明文检查、复杂运维中心

**服务自检接口**:
**公共发现服务** 暴露的健康检查接口分层。公开健康检查只能返回极简可用状态；详细诊断必须通过管理访问读取。
_Avoid_: 公开配置详情、公开密钥状态、单一 health 暴露全部

**容器健康检查**:
Docker Compose 用于判断 **公共发现服务** 基础可用性的检查。它覆盖进程响应、Postgres 连接、数据库迁移和 active 配置可读性，不检查 Steam、LLM、R2 或公共库是否为空。
_Avoid_: 第三方 API 抖动触发重启、空库视为不健康、泄露诊断详情




**同源管理托管**:
**公共发现服务** 主服务托管 **管理界面** 静态资源的部署形态。第一版管理界面与管理 API 同源，普通使用者客户端界面不从该服务加载。
_Avoid_: 独立管理前端部署、远程客户端 UI、客户端加载管理界面

**首次配置引导**:
**管理界面** 在 **公共发现服务** 尚未完成密钥配置时提供的初始化流程，用来收集 Steam 和 LLM 连接所需的密钥与非敏感参数。它写入服务端密钥配置介质，不写入数据库。
_Avoid_: 客户端引导、普通用户注册、数据库初始化

**首次配置必填项**:
**首次配置引导** 中创建可生成公共游戏库所需的最小配置。第一版 Steam Key 必填，LLM 与 R2 属于可跳过的增强或高级配置。
_Avoid_: 必填 LLM、必填 R2、无 Steam 生成公共库

**首次发现启动**:
首次配置完成并重启生效后，由维护者在管理界面明确触发的第一次公共库发现任务。第一版不在首次配置完成后自动开始发现。
_Avoid_: 配置后自动扫描、隐式消耗 API、客户端触发首次发现

**引导令牌**:
部署者在首次启动 **公共发现服务** 时提供的部署级凭据，用来进入 **首次配置引导**。首次配置完成后，引导令牌不再授予正常管理访问，但可作为安全修复模式的修复凭据。
_Avoid_: 管理员密码、客户端令牌、公开初始化入口

**修复凭据**:
**安全修复模式** 中用于进入配置修复流程的部署级凭据。第一版复用引导令牌作为修复凭据，但正常模式下引导令牌不能用于管理登录。
_Avoid_: 日常 admin token、无凭据修复、客户端凭据

**服务端密钥配置**:
**公共发现服务** 读取 Steam Key、LLM Key 和相关模型参数的服务端配置介质。第一版不把这些密钥写入数据库；管理界面可以更新该配置，并标记服务需要重启后生效。
_Avoid_: 数据库密钥、客户端密钥、本地用户配置

**第三方 API 密钥原文**:
Steam、LLM 和 R2 等第三方服务调用所需的可用密钥值。它们可以保存在受保护的服务端密钥配置文件中，但不能写入数据库、日志或在管理界面回显。
_Avoid_: 第三方 Key 哈希、WebUI 回显密钥、数据库 API Key

**配置生效状态**:
管理界面对配置变更是否已经生效的标记。第一版区分立即生效的热加载配置和需要服务重启后生效的配置。
_Avoid_: 保存即全部生效、隐藏待重启状态、配置状态不明

**配置状态记录**:
Postgres 中记录服务端配置版本、pending 状态、重启需求和启动结果的运维数据。它不包含配置文件全文、第三方 API 密钥或管理员令牌哈希。
_Avoid_: 配置文件入库、密钥入库、用数据库替代配置文件

**配置双槽**:
**公共发现服务** 管理服务端配置文件时使用的 active/pending 结构。管理界面写入 pending 配置，服务在重启前校验 pending，启动成功后再提升为 active。
_Avoid_: 直接覆盖 active、坏配置重启循环、无回退配置

**启动配置回滚**:
**公共发现服务** 启动时处理 pending 配置候选失败的回滚机制。配置回滚由服务启动逻辑完成，不依赖 Docker healthcheck 自动回滚。
_Avoid_: 容器自动回滚假设、失败后无限重启、丢失上一个 active 配置

**安全修复模式**:
**公共发现服务** 在 active 配置不可用或启动配置回滚失败时进入的最小修复状态。它只开放健康检查和受引导令牌保护的配置修复入口，不开放公共库 API、管理任务或第三方 API 调用。
_Avoid_: 坏配置后全功能运行、无凭据修复、任务继续执行



**密钥补丁配置**:
pending 配置中表达第三方 API 密钥变更的补丁或继承语义。管理界面不回显密钥，因此未修改的密钥必须继承 active 配置，不能因为保存非密钥配置而被清空。
_Avoid_: pending 完整复制密钥、未回显密钥被清空、隐式覆盖 secrets

**服务自重启**:
**公共发现服务** 在管理界面确认后通过优雅退出自身进程来触发容器运行时重启的机制。它依赖 Docker Compose restart policy 拉起服务，不使用重启代理，也不让服务控制 Docker API。
_Avoid_: Docker socket、重启代理、任意命令执行

**客户端界面**:
**使用者客户端** 中面向普通使用者的浏览和个人标记界面。它和 **管理界面** 是独立入口，只共享通用组件、类型和展示规则。
_Avoid_: 管理后台、配置后台、维护界面

**发现首页**:
**客户端界面** 的普通使用者第一屏，展示公共游戏库的摘要、分区预览和最近公开内容。它不是管理 dashboard。
_Avoid_: 管理概览、后台 dashboard、配置首页

**发现首页分区**:
**发现首页** 中面向普通使用者的公共库预览模块，例如新近公开、高置信推荐和最近入库。它只包含匿名可读的公共库内容，不展示管理任务、待审核数量、密钥状态或成本信息。
_Avoid_: 管理任务状态、后台失败项、配置状态卡


**空库客户端状态**:
**使用者客户端** 连接到空公共库时展示的只读状态，说明服务已连接但公共游戏库尚未生成。它不能提供 Steam、LLM 或发现任务配置入口。
_Avoid_: 客户端维护入口、客户端首次配置、普通用户导库

**匿名只读访问**:
**使用者客户端** 对 **公共发现服务** 的默认访问方式，只能读取公开游戏数据、服务身份信息和公开分析结果。它不能修改公共数据，不能触发发现任务，也不能读取或修改管理配置。
_Avoid_: 用户登录、访客账号、公开写入

**公开候选边界**:
匿名只读访问只能读取已接受且公开的游戏。待审核、已拒绝、隐藏和归档游戏只能通过管理访问查看。
_Avoid_: 候选池公开、拒绝项公开、普通客户端审核

**服务访问限流**:
**公共发现服务** 对匿名只读访问、管理登录、首次配置和一键重启施加的请求频率限制。管理、首次配置和重启相关接口必须比普通公共读接口更严格。
_Avoid_: 无限匿名访问、只靠前端按钮禁用、无保护管理入口

**公共读跨源访问**:
**公共发现服务** 对匿名只读 API 开放的可配置跨源访问能力，用于支持使用者客户端读取公共库数据。它不适用于管理、首次配置或重启相关 API。
_Avoid_: 跨源管理访问、跨源 setup、携带管理 cookie 的公共 CORS

**管理访问**:
维护者通过 **管理界面** 对 **公共发现服务** 执行写操作的访问方式，必须经过鉴权。它覆盖公共库维护、发现任务控制、AI 批处理和服务配置管理。
_Avoid_: 普通客户端访问、匿名管理、用户配置

**管理员令牌哈希**:
用于验证 **管理访问** 的管理员令牌派生值。第一版只保存哈希，不保存或回显管理员令牌明文。
_Avoid_: 明文管理员密码、可查看 token、数据库管理员密钥

**公共游戏库**:
由 **公共发现服务** 提供的共享多人游戏集合，包括游戏元数据、推荐分、AI 分析和发现状态。它不包含单个使用者的收藏、愿望单、关注或浏览记录。
_Avoid_: 个人游戏库、客户端数据库、用户库

**空公共库启动**:
新的 **公共发现服务** 首次启动时，Postgres 公共库可以为空。公共游戏库由首次配置后的 **服务端发现能力** 生成，而不是从旧本机库迁移。
_Avoid_: 预置正式库、旧库导入、客户端上传初始化

**公共库数据**:
Postgres 中支撑 **公共游戏库** 的数据集合，包含游戏、元数据、公开分析和面向匿名只读访问的查询模型。它与运维、任务和管理数据分属不同 schema。
_Avoid_: 管理数据、任务数据、个人状态

**游戏图片资源**:
**公共游戏库** 暴露给客户端展示的游戏图片引用。第一版默认使用经过服务端校验的 Steam CDN URL，并预留可选 R2 缓存 URL。
_Avoid_: 客户端任意图片 URL、强制图片代理、无来源图片

**图片缓存回填**:
启用 R2 图片缓存时，**公共发现服务** 在后台异步生成缓存图片 URL 的过程。图片缓存回填不能阻塞游戏发现、入库或公开展示。
_Avoid_: 阻塞式图片代理、无图不公开、客户端上传图片

**可选图片缓存配置**:
维护者为 R2 图片缓存提供的高级服务端配置。它不属于首次配置引导的必填项，也不写入数据库；未配置时公共游戏库继续使用原始 Steam CDN 图片。
_Avoid_: 必填 R2、数据库对象存储密钥、无 R2 不可用


**公共库当前态**:
**公共游戏库** 对普通客户端公开读取的最新游戏、元数据和分析结果。第一版只保留当前态和刷新记录，不保存完整历史快照。
_Avoid_: 完整版本历史、时间序列库、客户端合并历史

**游戏可见性**:
公共库当前态中决定游戏是否出现在普通客户端的展示状态，例如公开、隐藏或归档。游戏可见性不等同于物理删除。
_Avoid_: 删除状态、个人隐藏、客户端过滤

**游戏审核状态**:
维护者或规则流程对游戏是否适合进入公共游戏库的判断状态，例如已接受、待审核或已拒绝。它服务于公共库治理，不属于个人游戏状态。
_Avoid_: 用户评分、收藏状态、物理删除

**发现置信度分流**:
**服务端发现能力** 将发现结果按置信度分为自动公开、待审核或拒绝的规则。高置信多人游戏可以进入已接受且公开状态，中置信候选进入待审核，低置信或明显不匹配的候选进入拒绝。
_Avoid_: 全量自动公开、全量人工审核、客户端审核

**审核动作**:
维护者在 **管理界面** 中处理待审核游戏的操作，包括接受并公开、接受但隐藏、拒绝、归档和可选备注。第一版不包含多级审批、分配或评论线程。
_Avoid_: 客户端审核、多人审批、复杂审核流

**手动 AppID 添加**:
维护者通过 **管理界面** 输入 Steam AppID 并创建候选游戏的维护动作。它属于管理访问，仍需经过 Steam 拉取、规则分析和审核动作。
_Avoid_: 客户端提交、绕过审核、直接写公共库

**公开游戏分析**:
**公共游戏库** 中可被匿名只读访问读取的游戏分析内容，可以来自规则、LLM 或混合流程。它必须包含来源、生成时间、模型信息、评分版本和置信度等元信息。
_Avoid_: 内部 prompt、管理失败日志、无来源 AI 文案

**规则分析**:
不依赖 LLM Key 的基础游戏分析能力，用于生成推荐分、维度评分和可解释摘要。LLM 未配置或不可用时，公共游戏库仍应通过规则分析继续运行。
_Avoid_: AI 必需流程、无分析降级、客户端白屏

**LLM Key 清空**:
维护者在管理界面中移除 LLM Key 的敏感配置动作。它必须二次确认并重启生效；清空后新分析降级为规则分析，已有公开游戏分析仍可读取。
_Avoid_: 无确认清空、清空后分析不可用、删除已有分析

**历史 LLM 分析保留**:
LLM Key 清空或 LLM 不可用后，已经生成并公开的 LLM 分析继续保留其原始来源、模型和生成时间。系统不会自动删除或改写这些分析。
_Avoid_: 清空 Key 删除分析、自动改来源、历史分析失真

**分析重生成动作**:
维护者在管理界面中补齐缺失分析、重试失败分析或重生成选中游戏分析的操作。第一版不提供一键覆盖全部 LLM 分析的破坏性动作。
_Avoid_: 全量覆盖 LLM 分析、误删历史分析、无筛选重算

**运维管理数据**:
Postgres 中支撑 **管理访问** 和 **受控自动任务** 的数据集合，包含任务、运行记录、失败项、服务配置状态和审计日志。它不属于匿名只读访问的公共库数据。
_Avoid_: 公共游戏库、客户端状态、公开查询模型

**本地服务备份**:
维护者对 Postgres 数据和服务端配置文件执行的本地备份与恢复能力。第一版提供命令或脚本，不包含自动云备份。
_Avoid_: 云端自动备份、只备份数据库不备份配置、普通客户端备份

**服务迁移版本**:
**公共发现服务** 用来管理 Postgres schema 和服务端配置文件结构演进的版本标记。数据库迁移和配置文件迁移必须显式可追踪。
_Avoid_: 隐式迁移、无版本配置、手工改表

**服务错误信封**:
**公共发现服务** API 返回错误时使用的统一 JSON 结构，包含稳定错误代码、可读消息、请求 ID 和脱敏细节。公共 API 与管理 API 都使用该结构。
_Avoid_: 裸字符串错误、泄露密钥的 details、每个接口自定义错误形状

**错误消息语言**:
**服务错误信封** 中的错误代码使用英文稳定枚举，用户可读消息第一版使用中文。后续国际化应优先通过错误代码映射实现。
_Avoid_: 中文错误代码、前后端混合翻译、无稳定 code



**管理审计日志**:
**公共发现服务** 记录管理登录、首次配置、密钥配置更新、重启请求、任务触发和系统结果的审计数据。它不记录普通使用者的浏览行为或个人游戏状态。
_Avoid_: 用户行为追踪、密钥明文日志、客户端浏览日志

**脱敏失败摘要**:
**公共发现服务** 为第三方 API 调用失败保存的持久化错误信息，只包含阶段、目标、提供方、是否可重试、尝试次数和脱敏后的错误原因。默认不保存第三方 API 原始请求或原始响应。
_Avoid_: 原始 API 响应、密钥明文、完整 prompt 日志

**服务端发现能力**:
**公共发现服务** 拥有的公共数据维护能力，包括 Steam 同步、游戏发现、AI 分析、批处理、回填队列和自动调度。第一版这些能力不属于 **使用者客户端**。
_Avoid_: 客户端发现任务、本地同步任务、个人导库

**Steam 数据依赖**:
**公共发现服务** 生成或更新公共游戏库时对 Steam Key 和 Steam 数据源的依赖。Steam 不可用时，已有公共库仍可匿名只读访问，但服务端发现能力不能正常生成或更新公共数据。
_Avoid_: 客户端 Steam 配置、只读访问硬依赖、无 Key 生成公共库

**Steam Key 清空**:
维护者在管理界面中移除 Steam Key 的敏感配置动作。它必须二次确认并重启生效；清空后已有公共库可读，但生成或更新公共库的能力不可用。
_Avoid_: 无确认清空、客户端清空、清空后删除公共库

**受控自动任务**:
**公共发现服务** 按维护者配置自动运行的发现、同步或 AI 处理任务。它必须可关闭、可限额，并且保留管理界面中的手动触发入口。
_Avoid_: 无限制后台任务、客户端自动导库、隐式扫描

**任务限额配置**:
维护者在管理界面中设置的任务调度、请求上限、并发和成本保护配置。第一版自动任务默认关闭，并使用保守限额。
_Avoid_: 默认无限扫描、默认消耗 LLM、隐藏成本

**个人游戏状态**:
某个使用者对游戏的本地标记和行为状态，包括收藏、愿望单、关注和浏览记录。第一版 **个人游戏状态** 属于 **使用者客户端**，不属于 **公共发现服务**。
_Avoid_: 服务端用户偏好、云端收藏、账号数据

**个人状态迁移文件**:
**使用者客户端** 导出或导入个人游戏状态的本地文件。第一版只允许导入到相同服务实例 ID，不支持跨服务实例合并。
_Avoid_: 云同步、跨服务合并、公共库导入

**公共缓存导出**:
第一版已排除。个人状态迁移文件不包含公共库只读缓存、公开分析缓存或图片缓存。


## Flagged Ambiguities

**服务**:
已解析为 **公共发现服务**，不是让每个使用者自行部署的服务，也不是每台客户端内置的本地后台。

**游戏库**:
已拆分为 **公共游戏库** 与 **个人游戏状态**。公共数据来自 **公共发现服务**；个人标记第一版保存在 **使用者客户端**。

**默认服务地址**:
已排除。第一版允许修改 **服务地址**，但不提供默认官方服务地址，因此普通使用者仍需要输入或导入一次服务地址。

**连接文件鉴权**:
已排除。**服务连接文件** 只降低输入服务地址的成本，不代表登录或授权。

**系统协议连接**:
第一版已排除。使用者客户端不注册 `mpgs://` 系统协议；服务连接通过手动输入或导入服务连接文件完成。

**客户端候选提交**:
第一版已排除。普通使用者不能通过使用者客户端向公共发现服务提交或推荐 Steam AppID。

**连接验证**:
已限定为读取 **服务身份信息** 并检查 API 兼容性，不是登录，也不是对 Steam 或 LLM Key 的验证。

**客户端鉴权**:
第一版已排除。普通 **使用者客户端** 通过 **匿名只读访问** 使用公共数据；只有 **管理访问** 需要鉴权。

**客户端后端**:
已排除为第一版核心形态。**使用者客户端** 只保留 **客户端壳能力**，不会继续承载 **服务端发现能力**。

**客户端公共读代理**:
已排除。公共库读取由 **客户端界面** 直接执行客户端公共 API 调用，不能重新通过 Tauri Rust 命令包装成代理后端。

**复杂离线同步**:
已排除。第一版仅支持 **公共库只读缓存** 和本地 **个人游戏状态**，不做公共库写入、冲突合并或服务端个人状态同步。

**旧本机库迁移**:
已排除。服务端化视为项目翻新，现有本机 SQLite 中的公共游戏库数据不要求迁移到新的 Postgres 公共库。

**公共游戏物理删除**:
第一版默认排除。公共库治理使用 **游戏可见性** 和 **游戏审核状态**，而不是直接删除游戏记录。

**单一前端入口**:
已排除。**客户端界面** 和 **管理界面** 是同仓库内的两个独立入口，不能用一个应用入口承载两套权限语义。

**数据库密钥存储**:
已排除。Steam Key 和 LLM Key 属于 **服务端密钥配置**，不属于 Postgres 中的业务数据。

**第三方 API 原文日志**:
已排除。Steam 和 LLM 调用的原始请求、响应、Header、完整 URL query 和完整 prompt 默认不能进入持久化日志。

**单一数据库平铺表**:
已排除。第一版可以使用同一个 Postgres 数据库，但 **公共库数据** 与 **运维管理数据** 必须通过 schema 区分。

**主服务重启宿主容器**:
已排除。一键重启使用 **服务自重启**，不能让 **公共发现服务** 主进程直接拥有宿主容器控制权，也不引入重启代理。

**裸初始化入口**:
已排除。**首次配置引导** 必须先通过 **引导令牌**，不能在公网暴露无需凭据的初始化流程。

## Example Dialogue

开发者：朋友下载客户端后还需要填 Steam Key 吗？

领域专家：不需要。朋友只需要输入服务地址，让使用者客户端连接公共发现服务。

开发者：那谁负责发现新游戏和生成分析？

领域专家：公共发现服务负责。管理界面只给维护者使用，用来控制发现和分析流程，包括受控自动任务和手动触发。

开发者：新服务第一次启动时必须带一批旧游戏数据吗？

领域专家：不需要。新服务支持空公共库启动，首次配置完成后再由服务端发现能力生成公共游戏库。

开发者：客户端还会自己扫描 Steam 或调用 LLM 吗？

领域专家：第一版不会。使用者客户端只保留客户端壳能力；Steam 同步、发现任务和 AI 分析都属于服务端发现能力。

开发者：客户端读取公共游戏列表时要先调用 Tauri command 吗？

领域专家：不需要。客户端界面直接调用公共发现服务的匿名只读 REST API；Tauri command 只用于本地壳能力。

开发者：服务暂时连不上时，客户端还能看东西吗？

领域专家：可以看公共库只读缓存和本地个人游戏状态，但不能发现新游戏或更新公共分析。

开发者：朋友点收藏以后会同步到服务器吗？

领域专家：第一版不会。收藏、愿望单、关注和浏览记录都是个人游戏状态，保存在使用者客户端。

开发者：朋友填了一个服务地址，客户端能直接保存吗？

领域专家：不能。使用者客户端必须先读取服务身份信息，确认它是兼容的公共发现服务。

开发者：朋友不想复制服务地址怎么办？

领域专家：维护者可以发服务连接文件。客户端导入后仍会验证服务身份信息。

开发者：朋友看游戏列表需要登录吗？

领域专家：第一版不需要。朋友通过匿名只读访问查看公共游戏库；只有管理界面的写操作需要管理访问。

开发者：管理界面能直接塞进客户端界面吗？

领域专家：不能。客户端界面和管理界面是两个入口，它们可以共享组件和类型，但权限边界不同。

开发者：首次配置引导收集到的 Steam Key 会写进 Postgres 吗？

领域专家：不会。它属于服务端密钥配置，管理界面可以更新，但生效边界是服务重启。

开发者：服务第一次启动时，任何人都能打开初始化页面吗？

领域专家：不能。首次配置引导必须使用引导令牌进入，完成后引导令牌不再授予正常管理访问，只保留安全修复模式能力。

开发者：普通客户端连上空库服务后能开始配置 Steam Key 吗？

领域专家：不能。客户端只能展示空库客户端状态；公共库维护必须通过管理界面完成。

开发者：管理界面的一键重启是主服务自己操作 Docker 吗？

领域专家：不是。公共发现服务只会优雅退出自身进程，由 Docker Compose restart policy 重新拉起。
