# MPGS 客户端服务连接体验下一切片

- 文档日期：2026-06-09
- 适用范围：普通 Windows/Tauri 用户客户端的前端体验与客户端本地状态
- 依据文档：
  - `docs/2026-06-07-public-discovery-service-migration-roadmap.md`
  - `docs/2026-06-08-public-service-architecture-decisions.md`

## 1. 背景

当前实现已经完成公共发现服务的后端边界和客户端 REST 数据读取通道：

- 客户端已有服务地址验证模型：`src/domain/serviceConnection.ts`
- 客户端已有当前服务连接存储：`src/domain/serviceConnectionStorage.ts`
- 公共数据读取在存在服务连接时已走 REST：`src/api/publicServiceClient.ts`、`src/api/client.ts`
- 设置页已有“公共发现服务连接”区域，但入口较深，且主要依赖导入连接 JSON 文件

问题是普通客户端的用户体验仍接近旧的一体化本地应用：

- 首次进入仍容易看到本地 Steam/LLM 初始化语境
- 没有清晰的“输入服务器地址并连接”首屏
- 服务连接状态不够显眼
- 断开、重新验证、切换服务等操作不完整
- 公共服务模式下仍需要更强地隐藏或冻结本地发现、同步、AI 任务等旧能力入口

因此下一步不是继续扩展 admin-ui 或任务系统，而是完成普通客户端的“轻客户端连接体验”切片。

## 2. 目标

本切片目标是让普通客户端从用户视角明确成为一个连接公共发现服务的轻客户端。

完成后，用户应能：

1. 在没有服务连接时，首先看到“连接 MPGS 服务”的界面，而不是本地 Steam/LLM 初始化流程。
2. 直接输入服务地址并验证连接。
3. 继续支持导入维护者提供的无密钥连接文件。
4. 看到当前连接的服务名称、实例 ID、API 版本、公共库状态和最近验证时间。
5. 重新验证当前服务。
6. 断开当前服务连接，并回到连接界面。
7. 在公共服务模式下只看到普通用户需要的公共库浏览、个人收藏/愿望单/历史和设置能力。

## 3. 非目标

本切片不做：

- admin-ui 新功能
- 服务端任务系统新增能力
- Postgres schema 大迁移
- Steam/LLM/R2 配置迁移
- 多服务聚合
- 用户账号登录
- 公共库写入
- 客户端候选提交
- npm workspace 重组

## 4. 当前代码事实

### 已有能力

- `validateServiceAddress(baseUrl)` 已支持读取：
  - `GET /api/v1/service-info`
  - `GET /api/v1/discovery-home`
- 地址策略已包含：
  - 公网 HTTPS
  - localhost HTTP 开发例外
  - 可选局域网 HTTP 例外
- `saveCurrentServiceConnection()` 已保存当前服务连接。
- `clearCurrentServiceConnection()` 已存在，但 UI 层尚未形成完整断开流程。
- `getDashboard()` 在存在服务连接时会走公共 REST。
- 公共服务模式下已有部分隐藏逻辑，例如隐藏 AI 助手入口、隐藏本地维护控制。

### 主要缺口

- 没有专门的服务连接首屏。
- 设置页只提供导入连接文件，缺少直接输入服务地址。
- 公共服务模式下的设置页过于简略，缺少重验、断开、切换服务等操作。
- Onboarding 仍围绕本地 Steam/LLM 配置，不适合作为普通用户连接公共服务的首选入口。
- 顶部或侧边栏缺少当前数据源/连接状态的持久提示。

## 5. 设计要求

### 5.1 启动状态

启动时按以下顺序决策：

1. 如果存在已保存且格式兼容的服务连接，继续加载公共服务 dashboard。
2. 如果没有服务连接，显示服务连接界面。
3. 如果服务连接存在但公共读取失败：
   - 优先使用客户端只读缓存展示弱离线内容。
   - 如果无缓存，显示连接失败状态，并允许重新验证、修改地址、导入连接文件。

旧本地 onboarding 不应再作为普通用户默认第一入口。它可以保留为开发/兼容路径，但不能压过服务连接首屏。

### 5.2 服务连接界面

新增普通客户端连接界面，建议独立组件：

- `src/pages/serviceConnection/ServiceConnectionPage.tsx`

界面必须包含：

- 服务地址输入框
- 连接/验证按钮
- 导入连接文件入口
- 验证过程状态
- 验证失败诊断
- HTTPS/localhost/LAN 规则提示

直接输入地址时调用 `validateServiceAddress()`。验证成功后调用 `saveCurrentServiceConnection()`，再刷新 dashboard。

连接文件导入继续调用 `validateServiceConnectionFileText()`，不得跳过实时服务身份验证。

### 5.3 设置页连接管理

设置页“公共发现服务连接”应升级为完整连接管理区。

未连接时：

- 显示地址输入
- 显示导入连接文件
- 显示验证结果

已连接时：

- 显示服务名称
- 显示服务实例 ID
- 显示 API 版本
- 显示公共库状态
- 显示 Base URL
- 显示最近验证时间
- 提供重新验证按钮
- 提供断开连接按钮
- 提供切换服务入口

断开连接只清除客户端本地当前服务连接，不删除个人状态缓存。个人状态按 `serviceInstanceId` 分区保存，断开后仍可在重新连接同一实例时沿用。

### 5.4 公共服务模式 UI 收敛

当 `stats.sourceKind === "public_service"` 时：

- 隐藏本地 Steam Key 配置。
- 隐藏本地 LLM Key 配置。
- 隐藏本地发现任务、同步任务、AI 批处理入口。
- 隐藏本地 AI 推荐助手入口。
- 详情页分析区域保持只读。
- 设置页保留：
  - 服务连接管理
  - 个人状态说明
  - 缓存/重新读取相关操作
  - 关于信息

普通用户界面不展示管理任务、待审核、密钥、成本、队列或部署诊断。

### 5.5 连接状态提示

主界面应持续显示当前数据源状态，建议放在顶部栏或设置入口附近：

- 未连接
- 正在验证
- 已连接：服务名称
- 服务不可达：使用缓存或等待重试

状态提示只展示普通用户需要的信息，不泄露管理或部署细节。

## 6. 交互流程

### 6.1 新用户连接

1. 用户打开客户端。
2. 客户端发现没有当前服务连接。
3. 显示服务连接页。
4. 用户输入 `https://example.com`。
5. 客户端请求 `https://example.com/api/v1/service-info`。
6. 客户端验证 API 版本和 capability。
7. 客户端请求 `https://example.com/api/v1/discovery-home`。
8. 验证成功后保存连接。
9. 客户端加载发现首页。

### 6.2 导入连接文件

1. 用户选择 `mpgs-service-connection.json`。
2. 客户端检查 JSON 格式和敏感字段。
3. 客户端按文件中的 `baseUrl` 重新请求服务身份和公共读取接口。
4. 服务身份与文件一致时保存连接。
5. 客户端加载发现首页。

### 6.3 断开连接

1. 用户在设置页点击断开连接。
2. 客户端二次确认。
3. 客户端调用 `clearCurrentServiceConnection()`。
4. 客户端清空当前 dashboard 视图状态。
5. 客户端回到服务连接页。

## 7. 实施建议

建议分成 4 个小步骤：

1. 新增 `ServiceConnectionPage` 和直接地址验证表单。
2. 调整 `App.tsx` 启动模式：无连接时进入连接页。
3. 升级 `SettingsPage` 的服务连接管理能力。
4. 补齐公共服务模式 UI 隐藏逻辑和测试。

优先避免大规模重构。`OnboardingWizard` 当前有未提交改动，除非实现启动入口切换必须修改，否则先不改其内部流程。

## 8. 测试要求

至少补充以下测试：

- 无服务连接时渲染服务连接页。
- 输入 HTTPS 服务地址后调用 `/api/v1/service-info` 和 `/api/v1/discovery-home`。
- localhost HTTP 地址允许验证。
- 公网 HTTP 地址被拒绝。
- 导入连接文件仍会实时读取服务身份。
- 验证成功后保存连接并刷新 dashboard。
- 断开连接后回到服务连接页。
- 公共服务模式下不显示 Steam/LLM、本地发现、同步、AI 批处理入口。
- 有缓存但服务不可达时保留弱离线展示路径。

建议命令：

```powershell
npm test
npm run build
```

如果改动影响 Tauri 壳行为，再补充：

```powershell
npm run tauri -- build
```

## 9. 验收标准

本切片完成时必须满足：

- 普通用户不需要看到或配置 Steam Key、LLM Key 就能连接公共服务。
- 没有服务连接时，首屏明确要求连接 MPGS 服务。
- 用户可以直接输入服务地址连接，不必须依赖 JSON 文件。
- 已连接状态清晰可见。
- 用户可以重新验证和断开连接。
- 公共服务模式下旧本地服务端能力入口不可见。
- 公共数据读取继续走 REST API。
- Tauri Rust 侧没有新增公共库读取、发现、同步或 AI 任务职责。

## 10. 风险与边界

- 如果服务端公共库为空，客户端应显示“服务已连接，公共库尚未生成”的状态，而不是引导用户配置 Steam/LLM。
- 如果服务地址切换到不同 `serviceInstanceId`，个人状态不能误用到新服务。
- 如果只是服务地址变化但 `serviceInstanceId` 相同，应视为同一服务迁移地址。
- 断开连接不应删除按服务实例分区保存的个人状态。
- 本切片不改变 admin-ui 的同源管理模式。
