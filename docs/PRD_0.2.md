# MPGS 0.2 产品需求文档（M7：账号、社区与内容闭环）

| 字段 | 内容 |
| --- | --- |
| 产品版本 | `0.2.0` |
| 阶段 | M7 |
| 文档状态 | 待评审开发基线 |
| 基线日期 | 2026-07-17 |
| 前置条件 | M0–M6 工程基线；0.1 PRD 见 [PRD.md](PRD.md) |
| 目标用户 | 固定或半固定熟人游戏小组、关注新游和社区游玩意愿的玩家 |

## 1. 背景与问题

0.1 已建立推荐、搜索、详情、日历、反馈、想玩投票、AI 回退和桌面客户端，但真实用户体验仍存在以下断点：

1. 匿名身份依赖本地保存的令牌。清理浏览器/客户端数据后可生成新身份，投票数只能表示匿名身份数，不能稳定表示账户数。
2. 想玩投票已进入推荐排序和游戏详情，但缺少独立社区入口，其他用户难以发现并附和正在获得关注的游戏。
3. 即将发售推荐卡片没有展示发售日期，用户必须进入日历或详情才能看到。
4. AI 只能由服务端环境变量配置，用户无法选择内置 AI、自定义 OpenAI-compatible API 或关闭 AI。
5. 当前开发启动默认可进入演示数据模式；真实数据采集、持续富化、检索同步和媒体展示尚未形成可观察的产品闭环。
6. 之前的验收脚本曾把跳过步骤记为通过。M7 必须以真实产物、真实迁移和可观察用户流程作为退出证据。

## 2. 阶段目标

M7 必须完成以下闭环：

1. 建立账密账户和多设备会话，使社区写操作按账户唯一计数。
2. 提供“大家想玩”社区页面，展示投票趋势、累计人数和有限头像预览。
3. 在即将发售推荐流直接展示精度正确的发售日期。
4. 提供内置、自定义和关闭三种 AI 模式，并保证密钥安全。
5. 默认使用持久化真实数据，自动执行受控采集、富化和派生索引更新。
6. 用跨浏览器、跨进程、真实数据库和真实发布包完成验收。

## 3. 产品原则与已确认边界

- 匿名用户仍可浏览推荐、搜索、日历和详情；注册不是首屏门槛。
- 投票、反馈同步、跨设备偏好和 AI 自定义配置必须登录。
- “投票人数”定义为有效账户的去重数量，一个账户对一个游戏最多一票。
- 账密账户只能稳定标识“唯一账户”，不能证明一个账户对应唯一自然人。若以后需要真人级去重，应增加邮箱、手机号或 Steam OAuth 验证。
- “大家想玩”是独立社区视图，不改变既有四类推荐的产品语义。
- 确定性推荐仍是主路径；AI 关闭、超时或配置无效时必须回退。
- 密码、AI Key、刷新令牌和服务端密钥不得明文存储、返回或写入日志。
- 未知或模糊发售日期必须保持原精度，不得伪造成具体日期。
- 头像属于公开资料；上传和投票前必须明确告知其公开展示范围。

## 4. 核心用户流程

### 4.1 注册与登录

1. 匿名用户可正常浏览。
2. 用户首次执行“想玩”、反馈同步或保存跨设备设置时，客户端打开登录/注册对话框。
3. 注册输入用户名、显示名称和密码；头像可暂时使用系统生成默认头像。
4. 注册成功后，当前匿名身份的偏好、反馈和投票合并到新账户。
5. 登录成功后刷新访问令牌，加载账户头像、偏好、投票状态和 AI 设置。
6. 顶栏右侧显示账户头像；点击后可进入个人资料、AI 设置、退出当前设备或退出全部设备。

### 4.2 匿名数据合并

- 匿名偏好在账户没有云端偏好时迁移；两边都存在时以登录前明确确认的选择为准。
- 匿名反馈按幂等键迁移，重复事件不得再次写入。
- 匿名想玩票按 `(account_id, app_id)` 去重；合并后删除匿名票，不增加重复票数。
- 合并必须在单事务内完成，失败时匿名和账户数据均保持原状。
- 未登录的历史匿名票不进入 M7 公开账户投票人数，直到关联到有效账户。

### 4.3 大家想玩

1. 用户进入主导航中的“大家想玩”。
2. 默认展示“正在升温”，可切换“最多人想玩”。
3. 游戏条目展示名称、发售状态/日期、累计想玩人数和头像叠放预览。
4. 已登录用户可直接附和或取消；未登录用户点击后进入登录流程。
5. 投票成功采用乐观更新，服务端结果返回后校正数量和头像。
6. 用户进入详情时可看到同样的总数和更大的头像预览，但不提供完整用户目录。

### 4.4 查看即将发售游戏

1. 即将发售/Demo 卡片直接显示发售日期或“日期未定”。
2. 日期为月、季度或年份精度时显示模糊文本，例如“预计 2026 年第三季度”。
3. Demo、Playtest、抢先体验使用独立标签，不能与正式发售混淆。
4. 日期变更后，缓存 ETag 必须失效，并显示最近数据更新时间。

### 4.5 AI 设置

1. 用户在设置中选择“内置 AI”“自定义 API”或“关闭 AI”。
2. 内置 AI 显示可用状态、模型标签和账户额度，不显示服务端 Key。
3. 自定义 API 支持 OpenAI-compatible HTTPS 地址、模型和 API Key。
4. 用户保存前可测试连接；保存后只显示掩码状态，服务端不得再次返回原 Key。
5. 用户可替换或删除自定义 Key；删除后回到内置 AI或确定性模式。

## 5. 功能需求

### 5.1 账户与鉴权

| 编号 | 优先级 | 需求 | 验收摘要 |
| --- | --- | --- | --- |
| AUTH-001 | P0 | 用户名密码注册 | 用户名大小写不敏感且唯一；重复注册返回统一冲突错误 |
| AUTH-002 | P0 | 登录与令牌轮换 | 正确凭据返回短期访问令牌和可轮换刷新令牌 |
| AUTH-003 | P0 | 会话管理 | 支持当前设备退出、全部设备退出、刷新令牌复用检测 |
| AUTH-004 | P0 | 密码安全 | 使用 Argon2id、随机盐和版本化参数；数据库与日志无明文密码 |
| AUTH-005 | P0 | 密码修改 | 必须验证旧密码；修改后使其他刷新会话失效 |
| AUTH-006 | P0 | 登录防护 | 按账户标识与 IP 双键限流；错误信息不泄露账号是否存在 |
| AUTH-007 | P0 | 匿名身份合并 | 偏好、反馈和投票事务化迁移且去重 |
| AUTH-008 | P0 | 登录态持久化 | Tauri 使用操作系统安全凭据存储；Web 开发模式不得把长期令牌写入可读日志 |
| AUTH-009 | P0 | 账户删除 | 注销后立即失效会话、删除头像和公开投票；必要审计数据匿名化 |
| AUTH-010 | P1 | 密码找回 | 0.2 不提供无验证找回；后续接入邮箱或 Steam OAuth 后实现 |
| AUTH-011 | P0 | 顶栏账户入口 | 未登录显示“登录”；已登录显示头像菜单、资料、AI 设置和退出入口 |

用户名规则：`3–32` 个 ASCII 字母、数字或下划线，规范化为小写；显示名称 `1–40` 个 Unicode 字符。密码长度 `10–128`，不强制无意义的字符组合规则，但拒绝常见泄漏密码的本地阻断表可作为 P1。

### 5.2 用户资料与头像

| 编号 | 优先级 | 需求 | 验收摘要 |
| --- | --- | --- | --- |
| PROF-001 | P0 | 默认头像 | 未上传时生成稳定的颜色/首字头像，不依赖第三方服务 |
| PROF-002 | P0 | 上传头像 | 支持 JPEG、PNG、WebP；原文件不超过 2 MiB |
| PROF-003 | P0 | 图片处理 | 服务端解码后重编码为 128×128 WebP，裁剪为正方形并移除 EXIF |
| PROF-004 | P0 | 安全边界 | 不接收 SVG、HTML 或动图；验证真实内容类型而非只看扩展名 |
| PROF-005 | P0 | 公开范围 | 投票条目仅返回公开显示名称和头像 URL，不返回用户名、邮箱或内部令牌 |
| PROF-006 | P0 | 更新与删除 | 新头像立即使头像缓存版本失效；删除后回退默认头像 |
| PROF-007 | P1 | 审核能力 | 管理员可屏蔽违规头像并保留原因与审计记录 |

### 5.3 社区想玩页面

| 编号 | 优先级 | 需求 | 验收摘要 |
| --- | --- | --- | --- |
| COM-001 | P0 | 独立导航 | 增加“大家想玩”，不作为第五个推荐分区 |
| COM-002 | P0 | 账户级唯一票 | 只有有效账户可投票；同一账户同一游戏最多一票 |
| COM-003 | P0 | 正在升温 | 按最近 7 天新增账户票数排序，累计票数和最新投票时间作次级排序 |
| COM-004 | P0 | 最多人想玩 | 按累计有效账户票数排序，支持稳定游标分页 |
| COM-005 | P0 | 头像叠放 | 卡片最多展示 5 个头像；窄屏最多 3 个；其余显示 `+N` |
| COM-006 | P0 | 附和投票 | 复用乐观想玩按钮；成功后数量、头像与 ETag 同步更新 |
| COM-007 | P0 | 隐私与可访问性 | 头像具有显示名称替代文本；键盘与屏幕阅读器可获知“共 N 人想玩” |
| COM-008 | P0 | 缓存一致性 | 任意投票、账户注销或头像变更使相关社区页和详情缓存失效 |
| COM-009 | P0 | 反滥用 | 登录、设备与 IP 限流叠加；异常账户可冻结且不计入公开数量 |
| COM-010 | P1 | 过滤 | 可按发售状态、Demo、平台和人数过滤社区榜单 |

头像预览采用 facepile：当前用户已投票时优先显示当前用户，其余按最近有效投票时间选择，头像部分重叠但不得遮挡游戏标题、人数或投票按钮。`+N` 只表示省略数量，不可点击打开完整用户列表。

默认视觉规则：桌面头像 `28×28 px`、相邻重叠 `8 px`、最多 5 个；窄屏头像 `24×24 px`、最多 3 个；每个头像带与当前主题背景对比清晰的 `1 px` 描边，`+N` 与头像等高。最终尺寸可由主题 token 调整，但最大数量、溢出语义和无布局位移属于验收契约。

```text
┌─────────────────────────────────────────────┐
│ [封面]  Game Name                 2026-08-12 │
│         4 人合作 · 私人房 · Demo             │
│         ▲ 23 人想玩  (头像)(头像)(头像)(+20) │
└─────────────────────────────────────────────┘
```

### 5.4 发售日期与游戏媒体

| 编号 | 优先级 | 需求 | 验收摘要 |
| --- | --- | --- | --- |
| DATE-001 | P0 | Feed 日期字段 | 推荐条目返回 `release_date`、`release_date_raw`、`release_date_precision` |
| DATE-002 | P0 | 卡片日期 | 即将发售卡片直接显示精度正确的日期文本 |
| DATE-003 | P0 | 未定日期 | 无日期时明确显示“日期未定”，不参与具体日期排序 |
| DATE-004 | P0 | 日期变化 | 来源日期变化写历史事件并使 Feed/详情/日历 ETag 失效 |
| MEDIA-001 | P0 | 游戏封面 | 采集并展示官方商店胶囊图 URL、来源和更新时间 |
| MEDIA-002 | P0 | 媒体回退 | 图片缺失或加载失败时显示稳定占位，不改变卡片尺寸 |
| MEDIA-003 | P1 | 客户端缓存 | 缓存封面并限制容量；URL 仅允许批准的 Steam CDN 域名和 HTTPS |

### 5.5 AI 模式与用户配置

| 编号 | 优先级 | 需求 | 验收摘要 |
| --- | --- | --- | --- |
| AICFG-001 | P0 | 三种模式 | 内置、自定义、关闭；未配置时确定性推荐可用 |
| AICFG-002 | P0 | 内置额度 | 按账户执行日额度、并发上限和全局预算，设置页显示可用状态 |
| AICFG-003 | P0 | 自定义 Provider | 支持 OpenAI-compatible Base URL、模型和 API Key |
| AICFG-004 | P0 | 密钥安全存储 | 桌面端使用操作系统安全凭据库，浏览器仅保存当前标签页会话；服务端只保存自定义模式元数据，不保存用户 Key |
| AICFG-005 | P0 | 密钥不可回读 | 服务端 API 只返回模式、端点/模型元数据和本地配置状态，不返回原 Key |
| AICFG-006 | P0 | 连接测试 | 在保存/替换时执行有超时和响应大小限制的最小测试 |
| AICFG-007 | P0 | SSRF 防护 | 默认只允许 HTTPS 公网地址；拒绝私网、回环、链路本地和重定向绕过 |
| AICFG-008 | P0 | 缓存隔离 | AI 缓存键包含账户、Provider、模型、端点身份和提示词版本 |
| AICFG-009 | P0 | 删除配置 | 删除自定义密钥后不可恢复，后续请求回到内置或关闭模式 |
| AICFG-010 | P0 | 日志脱敏 | Key、Authorization、完整 Prompt 和用户原始查询不得进入普通日志 |

自定义 AI Key 只能通过已认证 HTTPS 请求进入服务端。单机桌面部署可使用 Tauri 安全桥接，但不得使用 `localStorage`、普通客户端 SQLite 或明文配置文件保存。

### 5.6 真实数据与自动更新

| 编号 | 优先级 | 需求 | 验收摘要 |
| --- | --- | --- | --- |
| DATA-201 | P0 | 持久化默认 | 非开发构建未配置持久化数据库时拒绝 ready，不自动进入演示模式 |
| DATA-202 | P0 | 演示模式显式化 | 只有显式 `MPGS_SEED_DEMO=true` 才能加载虚构数据，UI 显示“演示数据” |
| DATA-203 | P0 | 后台调度 | 服务启动后按配置运行候选采集、富化、质量检查和检索同步 |
| DATA-204 | P0 | 失败回退 | Steam 失败保留旧快照，记录重试和结构变化，不清空有效字段 |
| DATA-205 | P0 | 运行状态 | 管理状态接口返回上次成功、下次运行、错误类别、游标和数据覆盖率 |
| DATA-206 | P0 | 发布门禁 | 真实数据库满足候选、四分区、日期、封面和多人画像覆盖率后才能发布 |

## 6. API 契约增量

### 6.1 账户

```text
POST   /v1/auth/register
POST   /v1/auth/login
POST   /v1/auth/refresh
POST   /v1/auth/logout
POST   /v1/auth/logout-all
PUT    /v1/auth/password
GET    /v1/me
PATCH  /v1/me
DELETE /v1/me
PUT    /v1/me/avatar
DELETE /v1/me/avatar
```

### 6.2 社区

```text
GET  /v1/community/play-intents?sort=trending|most_voted&limit=&cursor=
POST /v1/games/{app_id}/play-intent
```

社区条目至少返回：

```json
{
  "app_id": 548430,
  "name": "Example",
  "release_date": "2026-08-12",
  "release_date_precision": "day",
  "play_intent": {
    "count": 23,
    "voted": true,
    "voters_preview": [
      { "display_name": "Player", "avatar_url": "/v1/users/u_x/avatar?v=3" }
    ],
    "omitted_count": 22
  }
}
```

`voters_preview` 不得包含登录用户名、邮箱、设备 ID、内部数据库行号或活动时间。

### 6.3 AI 设置

```text
GET    /v1/me/ai-settings
PUT    /v1/me/ai-settings
POST   /v1/me/ai-settings/test
DELETE /v1/me/ai-settings/custom-key
```

密钥写入接口成功后不得在响应、OpenAPI 示例、错误体或审计详情中回显。

## 7. 数据模型与迁移约束

优先复用现有 `anonymous_users.user_id` 作为主体 ID，并新增账户扩展表，避免重写偏好、反馈和投票的全部外键：

```text
user_accounts
  user_id PK -> anonymous_users.user_id
  username_normalized UNIQUE
  display_name
  password_hash
  password_scheme
  status
  created_at_ms / updated_at_ms

account_sessions
  session_id PK
  user_id
  refresh_token_hash UNIQUE
  device_label
  expires_at_ms / revoked_at_ms

user_avatars
  user_id PK
  version
  content_hash
  media_type
  storage_key
  updated_at_ms

user_ai_credentials
  user_id PK
  mode
  provider
  base_url
  model
  encrypted_api_key
  key_version
  updated_at_ms
```

迁移要求：

- 从每一个已发布 schema 版本升级到 M7 最新版本。
- 使用包含匿名偏好、反馈、想玩票和刷新令牌的代表性数据库验证升级。
- 迁移不得自动把未关联匿名票计入公开账户投票数。
- 备份恢复后密码哈希、加密 AI Key 和头像元数据必须保持一致；主密钥缺失时 AI 自定义配置不可用但普通推荐仍 ready。

## 8. 非功能与安全要求

| 编号 | 领域 | 0.2 要求 |
| --- | --- | --- |
| NFR2-001 | 鉴权性能 | 登录外普通鉴权 P95 增量小于 20 ms；密码哈希在独立阻塞线程池执行 |
| NFR2-002 | 社区性能 | “大家想玩”缓存命中 P95 小于 500 ms，首屏不做逐游戏 N+1 查询 |
| NFR2-003 | 密码安全 | Argon2id 参数满足部署基线并可版本升级；密码永不写日志 |
| NFR2-004 | 会话安全 | 访问令牌短期有效、刷新令牌轮换、撤销立即生效 |
| NFR2-005 | 头像安全 | 解码后重编码、限制像素/文件大小、固定响应类型、禁止 SVG |
| NFR2-006 | 密钥安全 | 客户端 Key 只进入设备安全凭据库或当前标签页会话；服务端不持久化原 Key，且 API 不可回读 |
| NFR2-007 | 隐私 | 公开 API 只暴露用户主动公开的显示名和头像；支持账户删除 |
| NFR2-008 | 可用性 | 账户、头像或 AI 子系统故障不得阻断匿名确定性推荐 |
| NFR2-009 | 可访问性 | 登录、头像、facepile、日期和投票支持键盘与屏幕阅读器 |
| NFR2-010 | 响应布局 | 1024×640、1280×800 和常用窄屏下头像、`+N`、日期和按钮不重叠 |

## 9. 产品指标定义

- `registered_accounts`：未删除、未冻结的账户数。
- `DAU`：UTC 自然日内至少完成一次已认证产品操作的去重账户数。
- `MAU`：滚动 30 天内至少完成一次已认证产品操作的去重账户数。
- `unique_voters(app_id)`：当前对游戏保持想玩状态的有效账户数。
- `vote_join_rate`：看到社区条目后新增想玩票的账户数 / 已登录曝光账户数。
- `anonymous_to_account_conversion`：注册账户数 / 触发登录提示的匿名主体数。
- `ai_builtin_usage`、`ai_custom_usage`、`ai_fallback_rate`：按账户和算法版本聚合，不记录原始 Key 或完整查询。

这些指标反映账户和产品活动，不得在产品文案中表述为“已验证真人数量”。

## 10. 验收场景

### 10.1 账户与迁移

1. 新用户注册、登录、刷新、退出和修改密码端到端通过。
2. 错误用户名与错误密码返回不可枚举的统一响应。
3. 同一账户在两个客户端登录，偏好和投票一致。
4. 清空一个客户端缓存后重新登录，不增加重复想玩票。
5. 匿名状态已有偏好、反馈和票，注册后合并且无重复。
6. 账户删除后全部会话失效，公开头像和票数更新。

### 10.2 社区与头像

1. 0、1、3、5、6、100 个投票账户分别验证头像和 `+N`。
2. 同一账户重复请求投票幂等，多个不同账户正确增加数量。
3. 取消投票、冻结账户和注销账户均使人数、头像和 ETag 更新。
4. 头像损坏、删除或加载失败时使用占位，不引发布局位移。
5. 社区榜单游标期间发生投票时返回明确的快照失效错误，不重复或漏项。

### 10.3 日期、AI 与数据

1. 日、月、季度、年份和 TBA 五种日期精度均有 UI 测试。
2. 即将发售卡片、日历和详情对同一游戏显示一致日期。
3. 内置 AI、自定义 AI、关闭、超时、额度耗尽和错误 Key 均有端到端测试。
4. 扫描客户端包、日志、错误体、OpenAPI 和数据库明文字段，不能出现测试密码或 AI Key。
5. 从真实数据库启动，四分区、社区、搜索、详情、日历、封面和 AI 回退均可操作。
6. Steam 断网后保留旧数据，并显示更新时间和采集失败状态。

## 11. 开发与发布门禁

吸取 M4–M6 的验收问题，M7 使用以下门禁：

1. 规格先行：数据库迁移、API DTO、错误码和 UI 状态必须在实现前确定。
2. 测试分层：领域单测、存储迁移、API 契约、Web 组件、Tauri 原生 E2E、发布包冒烟分别提供证据。
3. 真实数据门禁：演示种子只能验证交互，不能用于关闭 DATA-201～206。
4. 多主体门禁：至少使用 6 个独立账户验证去重、头像省略和社区排序，不能用同一浏览器缓存伪造多用户。
5. 安全门禁：密码哈希参数、令牌轮换、登录限流、头像解码、SSRF 和密钥脱敏必须有负向测试。
6. 产物门禁：CI 必须构建并运行实际服务包；跳过项只能记为 `SKIP`，不得计入 PASS。
7. 报告门禁：验收报告记录 Git SHA、工作树状态、迁移版本、API 契约摘要和包摘要。
8. 视觉门禁：桌面与窄屏截图检查 facepile、日期、登录对话框和 AI 设置，无文本遮挡或布局漂移。
9. 故障门禁：数据库锁、Steam/AI 超时、头像缺失、主密钥缺失和离线缓存均验证降级行为。

## 13. 非目标

- 本阶段不接入 Steam 登录、好友列表或游戏库交集。
- 不提供用户私信、群组、评论区、帖子或关注关系。
- 不以账密账户宣称完成真人实名或反女巫证明。
- 不建立完整内容审核平台；仅提供头像屏蔽和审计入口。
- 不训练基于社区投票的 Learning-to-Rank 模型；投票仍是有界信号。
- 不在没有邮箱/Steam OAuth 的情况下提供不安全的密码找回。
