# 隐私政策草案（MVP 0.1）

**状态**：工程基线文案，供发布负责人合规签字；上线主体与联系邮箱在签字时填入。
**适用产品**：MPGS 桌面客户端与配套 `mpgs-server`（熟人联机游戏推荐）。

## 1. 我们收集什么

MVP 以**匿名可用**为设计目标，不要求 Steam 账号或真实姓名。

| 类别 | 内容 | 目的 |
| --- | --- | --- |
| 设备/会话标识 | 随机设备 ID、匿名会话令牌 | 偏好、反馈、限流与 ETag 缓存 |
| 偏好 | 人数、合作/竞技倾向、平台、预算、时长、语言等 | 个性化推荐 |
| 反馈与游玩意愿 | 喜欢/不感兴趣等原因、想玩投票 | 改进排序与产品指标 |
| 自然语言查询 | 用户主动输入的推荐文本 | 意图解析与检索（可关闭 AI） |
| 技术日志 | 请求 ID、路由、状态码、耗时、错误类别 | 可靠运行与排障 |
| 客户端缓存 | 推荐快照、详情、待同步反馈 | 离线浏览与断线重放 |

游戏详情还会保存并展示 Steam 已公开的热门评价正文、公开昵称及个人资料链接。这些内容属于外部公开目录数据，不用于识别 MPGS 用户，也不会把评论作者的 SteamID 暴露给客户端。

## 2. 我们不收集什么

- Steam 密码、Steam Guard、钱包或支付信息。
- 非必要的精确通讯录、麦克风、摄像头。
- 将客户端作为通用 SQL 或任意文件探针的数据。

Steam Web API Key 与内置 AI Key **仅存在于服务端环境**，不进入客户端包。用户自定义 AI Key 仅保存在其设备：桌面端使用操作系统凭据库，浏览器预览使用当前标签页会话存储；调用时临时发送给 MPGS 后端转发，但不写入服务端数据库、缓存或日志。

## 3. 处理与存储

- 权威数据位于**与服务同机的 SQLite**；客户端仅有独立缓存库。
- 可选 AI 调用会将**检索候选摘要与用户查询**发送至配置的 Provider；关闭 AI 时走确定性路径。
- 备份由运营方按 [OPERATIONS.md](OPERATIONS.md) 管理；恢复流程见 [ROLLBACK.md](ROLLBACK.md)。

## 4. 保留与删除

- 匿名会话与刷新令牌具有过期时间并轮换。
- 用户可在客户端清理缓存（默认保留未同步反馈）。
- 运营方删除用户行/会话的运维流程在内测阶段以数据库作业完成；正式主体应提供联系渠道。

## 5. 共享

- 不出售个人数据。
- 仅在提供 AI 增强且用户发起自然语言推荐时，向已配置的 AI 供应商发送完成该请求所需的最小上下文。
- Steam 商店跳转在用户设备上打开标准商店链接，不经过 MPGS 代理登录。

## 6. 安全

- 管理 API 使用独立 Bearer 令牌与限流。
- 传输层建议由部署方 TLS 终止。
- 日志禁止写入 Key、令牌与完整 Prompt。

## 7. 儿童

产品面向通用桌面用户；不主动收集儿童个人数据。若当地法律要求年龄门槛，由发布主体补充。

## 8. 变更

政策变更将提高文档版本并在发布说明中提示。继续使用新版本客户端/服务即表示知悉更新（最终法律效力以签字主体文本为准）。
